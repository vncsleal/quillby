# Security Policy

## Supported versions

This project is currently maintained on the latest `main` branch.

## Reporting a vulnerability

If you discover a security issue, please do not open a public issue with exploit details.

Instead:
1. Open a private report via GitHub Security Advisories (preferred), or
2. Open an issue with minimal details and request a private follow-up.

Include:
- Affected area/files
- Reproduction steps
- Potential impact
- Suggested mitigation (if any)

We aim to acknowledge reports within 72 hours.
